# denote-scribe

`denote-scribe` turns a completed AI conversation into an Org-mode Denote report saved under `~/Dropbox/notes`.

Trigger examples:

- `输出denote报告`
- `生成denote报告`
- `save this chat as a Denote report`

## Requirements

- `emacsclient` available on `PATH`
- A running Emacs server
- Denote available in that Emacs session
- A notes directory, defaulting to `~/Dropbox/notes`

The skill provides an Emacs Lisp entry point:

```elisp
(load-file "/path/to/denote-scribe/scripts/denote-scribe.el")
(denote-scribe-create "Denote技能改造" "/tmp/report.org" '("ai" "report") "~/Dropbox/notes/")
```

The title is generated by the AI from the solved problem. It should be short and specific, not a generic title like `问题处理报告`.

For Codex or Claude, call the same Elisp through `emacsclient --eval` after generating the temporary Org report body.

The skill itself contains no shell wrapper script; Denote integration lives in `scripts/denote-scribe.el`.

## Install In Codex

Copy or symlink this directory into your Codex skills directory:

```bash
mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
cp -R denote-scribe "${CODEX_HOME:-$HOME/.codex}/skills/"
```

Restart Codex or reload skills. After that, say `输出denote报告` after a task is complete, or invoke it explicitly with `$denote-scribe`.

## Install In Claude

Claude skills use the same folder shape: a skill directory with `SKILL.md` at the root and optional resource folders.

To install manually, add this `denote-scribe` directory to the skills location used by your Claude client or workspace. If your Claude client supports importing a skill archive, zip the directory and import it:

```bash
zip -r denote-scribe.zip denote-scribe
```

Then enable the skill in Claude. After a task is complete, say `输出denote报告` and Claude should summarize the conversation and call `denote-scribe-create` from `scripts/denote-scribe.el`.

## Notes

The Elisp intentionally calls Denote rather than only writing a filename that looks like Denote.
