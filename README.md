# denote-scribe

`denote-scribe` turns a completed AI conversation into an Org-mode Denote report saved under `~/Dropbox/notes`.

Trigger examples:

- `输出denote报告`
- `生成denote报告`
- `save this chat as a Denote report`

## Requirements

- `emacsclient` available on `PATH`
- A running Emacs server
- Denote available in that Emacs session
- A notes directory, defaulting to `~/Dropbox/notes`

The skill provides an Emacs Lisp entry point:

```elisp
(load-file "/path/to/denote-scribe/scripts/denote-scribe.el")
(denote-scribe-create "Denote技能改造" "/tmp/report.org" '("ai" "report") "~/Dropbox/notes/")
```

The title is generated by the AI from the solved problem. It should be short and specific, not a generic title like `问题处理报告`.

For any AI assistant, call the same Elisp through `emacsclient --eval` after
generating the temporary Org report body.

The skill itself contains no shell wrapper script; Denote integration lives in
`scripts/denote-scribe.el`.

## Install

Install this directory in the skills location used by your client or workspace.
The common shape is a skill directory with `SKILL.md` at the root and optional
resource folders such as `scripts/` and `agents/`.

To install manually, copy or symlink the skill directory into that location. If
your client supports importing a skill archive, zip the directory and import it:

```bash
zip -r denote-scribe.zip denote-scribe
```

After enabling or reloading the skill, say `输出denote报告` after a task is
complete. The assistant should summarize the conversation, load
`scripts/denote-scribe.el`, and call `denote-scribe-create`.

## Notes

The Elisp intentionally calls Denote rather than only writing a filename that looks like Denote.
